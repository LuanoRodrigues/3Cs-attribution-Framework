Scope and Objective

This methodology formalizes cyber-attribution assessment as a claim-based evidentiary scoring process. Each claim is scored on calibrated C-axes and then aggregated to document-level indicators under explicit weighting assumptions.

Execution mode: aggregate.

Pipeline and Data Lineage

The pipeline executes deterministic stages: PDF to markdown, schema extraction, source inference, validation, scoring, and figure rendering. Each stage emits machine-readable artifacts for auditability.

Validation Gates and Failure Semantics

Validation checks schema conformance, anchor consistency, and grounding quality. FAIL denotes a quality gate failure, not necessarily runtime interruption. The pipeline can proceed in fail-soft mode to preserve diagnostic visibility.
